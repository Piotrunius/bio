## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the existing issues to avoid duplicates. When you create a bug report, include as many details as possible:

- **Use a clear and descriptive title**
- **Describe the exact steps to reproduce the problem**
- **Provide specific examples** to demonstrate the steps
- **Describe the behavior you observed** and what you expected to see
- **Include screenshots or animated GIFs** if possible
- **Include your browser version and operating system**

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion:

- **Use a clear and descriptive title**
- **Provide a detailed description** of the suggested enhancement
- **Explain why this enhancement would be useful**
- **List some examples** of how it would be used

### Pull Requests

1. Fork the repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes (`git commit -m 'Add some amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

#### Pull Request Guidelines

- **Keep it focused**: One feature/fix per PR
- **Write clear commit messages**: Explain what and why, not just how
- **Test your changes**: Make sure everything works as expected
- **Update documentation**: If you change functionality, update the README
- **Follow the existing code style**: Maintain consistency with the existing codebase
- **Keep it small**: Smaller PRs are easier to review and merge

## Code Style Guidelines

### JavaScript

- Use ES6+ features where appropriate
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused
- Use `const` by default, `let` when reassignment is needed
- Avoid `var`

### CSS

- Use CSS custom properties (variables) for colors and common values
- Follow BEM naming convention where applicable
- Keep selectors specific but not overly complex
- Group related styles together
- Add comments to separate major sections

### HTML

- Use semantic HTML5 elements
- Include proper ARIA labels for accessibility
- Keep markup clean and well-indented
- Use meaningful class names

## Accessibility

This project values accessibility. Please ensure your contributions:

- Work with keyboard navigation
- Include proper ARIA labels
- Have sufficient color contrast
- Work with screen readers
- Support reduced motion preferences

## Performance

Consider performance when making changes:

- Minimize DOM manipulations
- Use event delegation where appropriate
- Optimize images and assets
- Lazy load non-critical resources
- Test on mobile devices and low-end hardware

## Testing

While this project doesn't have automated tests (yet), please:

- Test your changes in multiple browsers (Chrome, Firefox, Safari, Edge)
- Test on mobile devices
- Verify keyboard navigation works
- Check that dark/light theme both work
- Ensure no console errors appear
